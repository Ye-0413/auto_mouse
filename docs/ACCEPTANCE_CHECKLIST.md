# Anything Auto — MVP 验收清单

**文档用途：** 对照 `docs/DEVELOPMENT_PLAN.md` §3 MVP 成功标准与本仓库已实现能力，做一次可勾选的手工与自动化验收。  
**维护：** 功能变更时请同步更新对应条目。

---

## 执行摘要（Executive summary）

| 维度 | MVP 范围内 | MVP 范围外（本清单仍可能提及） |
|------|------------|-------------------------------|
| **目标用户** | Windows 桌面批量操作（Excel 驱动） | 多用户、云端调度、权限体系 |
| **核心闭环** | Excel → 列映射/预览 → 选流程 → 批量执行 → 日志/导出 | 拖拽式可视化设计器、全自动 AI 修流程 |
| **技术** | PySide6 GUI、SQLite、`FlowRunner`、桌面自动化（pyautogui 等）、可选 Playwright、`if` 条件 DSL（无裸 `eval`） | 完整 OCR/图像模板体系、大规模并发 |
| **录制** | 本机可选 `pynput` 录制：点击 / 打字合并 / 快捷键 → 保存或追加流程 | 高级语义录制、跨机器回放保证 |

---

## 如何使用本清单

- 每条包含：**如何验证**、**期望结果**、**Owner / 工具**（可填姓名或「QA」「Dev」）。
- `[ ]` 为手工或一次性步骤；自动化已由现有测试覆盖的项在「工具」中标明 `pytest`。

## 5. 流程能力与 Runner（对齐 §3-4）

对下列步骤类型：**至少各手工执行一次**或在合成流程中串联（可与批量页结合）。

- [ ] **`activate_window`**（Windows）  
  - **验证：** 标题包含匹配激活目标窗口。  
  - **期望：** 目标窗口前台（或非 Windows 跳过提示，与实现一致）。  

- [ ] **`open_url` / `open_file`**  
  - **验证：** 打开合法 URL、本地文件或文件夹。  
  - **期望：** 系统默认程序响应且无 Runner 致命错误。  

- [ ] **点击 / 输入：`click_mouse`、`input_text`、`hotkey`、`paste_clipboard`**  
  - **验证：** 构造最小步骤列表执行。  
  - **期望：** 屏幕坐标点击、文本输入、组合键、粘贴行为符合 `desktop.run_step` 实现。  

- [ ] **`wait`**  
  - **验证：** 设定毫秒等待。  
  - **期望：** 顺序节奏符合设定。  

- [ ] **`if` 分支 + 安全条件 DSL（禁止裸 eval）**  
  - **验证：** 配置 then/else 子步骤各至少一步；命中不同分支。  
  - **期望：** 条件解析与分支执行正确；无 `eval`。  
  - **自动化：** `pytest tests/test_condition_engine.py`  

- [ ] **`note` 跳过执行**  
  - **验证：** 流程中含 `note`。  
  - **期望：** 不调用桌面自动化适配器。  
  - **自动化：** `pytest tests/test_runner.py`（含 note 用例）
